정적 언어 vs 동적 언어

정적 언어
- 컴퓨터에 대한 저수준의 세부사항을 몇가지 정할 필요가 있기 때문에 정적 언어라고 불린다.

정적 언어의 특징
1. 정적 언어에서는 각 변수의 타입을 선언한다(변수 타입은 메모리에서 사용할 공간과 용도를 컴퓨터에 알려준다).

2. 정적언어라고 불리우는 이유는 변수는 자신의 타입을 변경할 수 없다. 즉 한 번 정수는 영원한 정수이다.

동적 언어의 특징
1. 변수 타입을 선언하도록 강요하지 않는다(ex. x = 5와 같이 입력하면 동적 언어는 변수 x가 선언되고, 그 값이 정수 5인지를 알고 있다).

2. 코드를 컴파일러 대신 인터프리터라는 프로그램으로 해석한다.

3. 정적 언어보다 속도가 느리다(but 인터프리터가 최적화되면서 동적 언어의 속도가 향상되고 있다).

4. 오랜시간 동안 동적 언어는 정적 언어로 짠 프로그램의 데이터 공급을 위해 코드가 짧은 프로그램(스크립트)로 사용되었다(이를 glue코드라고 부른다). => 오늘날에는 아주 큰 업무 처리 수행도 잘함.

Perl => 광범위한 라이브러리를 가지고 있지만 사람에 따라 구문이 어색하게 느껴질 수 있어서, 최근 몇 년동안 인기를 잃음

Ruby => 펄의 특징을 조금 빌린 최신 언어. Ruby on Rails라는 웹 프레임워크 때문에 큰 인기를 얻었음.

PHP => HTML과 쉽게 결합할 수 있어서 웹 개발에서 아주 큰 인기를 얻음. BUT 많은 사람이 PHP 언어를 알지만, 웹 이외의 일반 언어로는 인식하지 않는다.

파이썬을 사용하는 이유
- 파이썬은 범용의 고수준 언어이며 코드를 읽기 쉬움. 이는 배우기 쉽다는 뜻. => 코드를 작성하기도 쉬움.

파이썬을 사용하면 안 될 때
- 프로그램이 계산 작업을 많이 하는 경우 => 일반적으로 C, C++, JAVA로 작성한 프로그램은 파이썬으로 작성한 프로그램보다 빠르다.
